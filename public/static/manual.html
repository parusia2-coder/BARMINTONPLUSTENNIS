3. 네트워크 구성

3-1. 체육관 Wi-Fi 환경의 문제점

배드민턴 체육관은 일반 실내 공간과 달리 Wi-Fi에 매우 불리한 환경입니다.

문제 요인상세 설명대응 방법
넓은 공간 일반 체육관 40m×30m 이상. 가정용 공유기 실내 유효 거리 10~15m에 비해 훨씬 넓어 코트 끝에서 신호가 약해짐 공유기를 코트 중앙 높은 곳에 설치
금속 구조물 철골 지붕, 철제 관중석, 금속 칸막이 등이 전파를 반사/차단하여 사각지대 발생 공유기 2대로 커버리지 확장
다수 동시 접속 참가자 177명 + 운영 장비 10여대가 동시에 접속. 저가 공유기는 동시 접속 10~15대에서 성능 저하 동시 접속 50대+ 지원 공유기 사용
2.4GHz 간섭 참가자 스마트폰, 블루투스 이어폰, 전자레인지 등이 2.4GHz 대역에 몰려 혼잡 발생 코트 태블릿은 반드시 5GHz 대역 사용

3-3. 5GHz vs 2.4GHz 대역 분리

대부분의 듀얼밴드 공유기는 2개의 Wi-Fi 이름(SSID)을 동시에 방송합니다. 이 두 대역을 용도별로 나눠 사용하면 서로 간섭 없이 안정적으로 운영할 수 있습니다.

구분5GHz 대역2.4GHz 대역비교 설명
Wi-Fi 이름 예시 BADMINTON-5G BADMINTON 공유기 설정에서 SSID를 다르게 지정
속도 빠름 (최대 1,200Mbps) 보통 (최대 300Mbps) 5GHz가 약 2~4배 빠름
도달 거리 짧음 (10~20m) 김 (20~35m) 2.4GHz가 벽/장애물 통과 유리
간섭 적음 많음 (블루투스, 전자레인지 등) 5GHz 채널이 훨씬 여유로움
연결 장비 코트 태블릿 (6대)

💡 대역 분리가 중요한 이유

참가자 177명이 모두 BADMINTON-5G에 접속하면 코트 태블릿의 점수 전송이 지연될 수 있습니다. 핵심 운영 장비(태블릿, 본부)는 5GHz 전용으로, 참가자 스마트폰은 2.4GHz로 분리하면 점수 입력이 끊김 없이 안정적으로 동작합니다.

📱 태블릿 Wi-Fi 연결 방법 (대역 지정)
iPad:
  1. 설정 → Wi-Fi
  2. BADMINTON-5G 선택 (5G 붙은 이름)
  3. 비밀번호 입력 후 연결
  4. BADMINTON(2.4GHz)는 "이 네트워크 지우기"로 자동연결 방지
안드로이드 태블릿:
  1. 설정 → 연결 → Wi-Fi
  2. BADMINTON-5G 선택
  3. 비밀번호 입력 후 연결
  4. BADMINTON(2.4GHz)는 길게 눌러 "삭제"

3-7. 인터넷 회선 확보 방법

방법장점단점적합한 상황
① 체육관 유선 인터넷 가장 안정적, 속도 빠름 사전 확인 필요, LAN 포트 위치 제한 최우선 권장
② LTE/5G 에그 (포켓와이파이) 어디서든 사용, 별도 회선 불요 속도 불안정, 데이터 제한, 동시 접속 10대 내외 유선 없는 체육관
③ 스마트폰 테더링 긴급 백업, 별도 장비 불필요 배터리 소모 빠름, 동시 접속 5~10대 비상용 백업 전용
🔴 인터넷 회선 사전 확인 (대회 1주 전)
  • 체육관에 유선 인터넷(LAN 포트)이 있는지 관리사무소에 확인
  • LAN 포트 위치 확인 → 공유기까지 LAN 케이블 길이 산정 (여유분 5m 추가)
  • 유선 인터넷 없는 경우: LTE 에그 렌탈 또는 데이터 무제한 요금제 스마트폰 준비
  • 현장 답사 시 스마트폰으로 LTE/5G 신호 강도 확인 (지하 체육관은 약할 수 있음)

인터넷 끊김 감지 (태블릿에 "오프라인" 경고 표시) │ ├─ 1단계: 공유기 전원 껐다 켜기 (30초 대기) │ → 복구되면 모든 장비 자동 재연결 │ ├─ 2단계: 공유기 복구 안 됨 │ → 백업 스마트폰 핫스팟 ON │ → 핫스팟 SSID/비밀번호를 공유기와 동일하게 설정하면 │ 장비 재연결 없이 자동 전환 (사전 설정 권장) │ └─ 3단계: 인터넷 자체 불가 (지하 체육관 등) → 수기 기록으로 전환 (점수 용지 비치) → 대회 종료 후 시스템에 일괄 입력
💡 핫스팟 백업 사전 설정 (권장)

대회 전날에 백업용 스마트폰의 핫스팟 이름(SSID)과 비밀번호를 공유기와 동일하게 설정해두면, 공유기 장애 시 핫스팟만 켜면 태블릿이 자동으로 연결됩니다. 장비를 하나하나 재설정할 필요가 없어 30초 이내에 복구됩니다.

iPhone 핫스팟 이름 변경:
  1. 설정 → 일반 → 정보 → 이름
  2. 공유기 SSID와 동일하게 변경
  3. 설정 → 개인용 핫스팟 → 비밀번호를 공유기와 동일하게
안드로이드 핫스팟 이름 변경:
  1. 설정 → 연결 → 모바일 핫스팟
  2. 네트워크 이름을 공유기 SSID와 동일하게
  3. 비밀번호를 공유기와 동일하게

3-9. 당일 아침 네트워크 테스트

순서테스트 항목방법합격 기준
① 인터넷 연결 확인 공유기 연결 후 스마트폰으로 네이버 접속 3초 이내 페이지 로드
② 5GHz 신호 강도 가장 먼 코트 태블릿에서 Wi-Fi 신호 확인 신호 막대 2칸 이상
③ 속도 테스트 가장 먼 코트에서 브라우저 → fast.com 접속 다운로드 5Mbps 이상
④ 시스템 접속 확인 각 태블릿에서 코트 URL 접속 5초 이내 화면 표시
⑤ 점수 전송 테스트 태블릿에서 테스트 점수 입력 → 본부에서 반영 확인 2초 이내 반영
⑥ 동시 접속 테스트 운영진 5~6명이 동시에 다른 페이지 접속 모든 장비 정상 로드
✅ 테스트 통과 후

테스트 점수 데이터 원복 (본부에서 점수 0으로 수정) → 네트워크 정보를 체크리스트 메모란에 기록

기록할 정보: 5GHz SSID, 2.4GHz SSID, Wi-Fi 비밀번호, 공유기 관리자 IP (보통 192.168.0.1)

3-10. 사설 IP 로컬 서버 운영 (인터넷 없는 환경) 중요

인터넷 회선이 없는 체육관에서도 공유기 + 노트북 + 태블릿만으로 대회를 운영할 수 있습니다. 노트북에서 로컬 서버를 실행하고, 태블릿은 사설 IP로 접속합니다.

🌐 네트워크 구성도 (사설 IP)
┌──────────────────────────┐ │ Wi-Fi 공유기 (AP) │ │ 게이트웨이: 192.168.0.1 │ │ DHCP: 192.168.0.100~254│ └──────┬─────────┬──────────┘ │ │ ┌──────────┘ └──────────┐ ▼ ▼ ┌─────────────────┐ ┌─────────────────┐ │ 💻 노트북 │ │ 📱 태블릿 1~6 │ │ 192.168.0.100 │ │ 192.168.0.201~ │ │ (로컬 서버) │ │ (코트별 점수판) │ │ :3000 포트 │ │ │ └─────────────────┘ └─────────────────┘ 태블릿 접속 URL: http://192.168.0.100:3000

① 공유기 기본 설정

순서설정 항목방법
1 공유기 관리 페이지 접속 노트북을 공유기에 Wi-Fi 연결 → 브라우저에서 192.168.0.1 접속 (제조사별 다름: ipTIME 192.168.0.1, TP-Link 192.168.0.1 또는 tplinkwifi.net)
2 DHCP 범위 설정 고급 설정 → 네트워크 관리 → 내부 네트워크 설정
• 시작 IP: 192.168.0.100
• 끝 IP: 192.168.0.254
• 서브넷 마스크: 255.255.255.0
• 게이트웨이: 192.168.0.1
3 노트북 IP 고정 (권장) 고급 설정 → DHCP → 수동 IP 할당
• 노트북의 MAC 주소를 찾아 192.168.0.100 고정 할당
※ MAC 주소 확인법: Windows → ipconfig /all → 물리적 주소, Mac → 시스템 설정 → Wi-Fi → 상세 정보 → MAC 주소
4 Wi-Fi 이름/비밀번호 설정 무선 설정에서 SSID를 BADMINTON-5G(5GHz), BADMINTON(2.4GHz)로 설정
보안: WPA2-PSK, 비밀번호를 간단하게 (예: badminton2026)
5 설정 적용 저장 후 공유기 재부팅 (30초 대기)

② 노트북 IP 확인 및 고정

🪟 Windows 노트북

IP 확인:

Win + R → cmd → Enter ipconfig

무선 LAN 어댑터 → IPv4 주소 확인 (예: 192.168.0.100)

수동 고정 IP 설정 (DHCP 고정 안 할 경우):

  1. 설정 → 네트워크 및 인터넷 → Wi-Fi → 하드웨어 속성
  2. IP 할당 → 편집 → 수동
  3. IPv4 켜기:
    • IP 주소: 192.168.0.100
    • 서브넷 마스크: 255.255.255.0
    • 게이트웨이: 192.168.0.1
    • DNS: 8.8.8.8
  4. 저장
🍎 Mac 노트북

IP 확인:

시스템 설정 → Wi-Fi → 연결된 네트워크 클릭 → "상세 정보" 또는 터미널: ifconfig en0 | grep inet

수동 고정 IP 설정:

  1. 시스템 설정 → Wi-Fi → 연결된 네트워크 → 상세 정보
  2. TCP/IP 탭 → IPv4 구성: 수동
  3. 설정:
    • IP 주소: 192.168.0.100
    • 서브넷 마스크: 255.255.255.0
    • 라우터: 192.168.0.1
  4. DNS 탭 → 8.8.8.8 추가
  5. 확인

  2. 하단 공유 버튼 (□↑) 탭 → "홈 화면에 추가"

  4. 이름 입력 → 조회 → "경기 시작 알림 받기" 클릭

  • 테스트 발송: 알림 구독 후 "테스트" 버튼으로 정상 수신 확인 가능

⚡ 일괄 생성 기능 (종목/팀 탭)

종목/팀 탭의 보라색 "일괄 생성" 버튼을 클릭하면 4단계 모달이 열립니다:

STEP 1 STEP 2 STEP 3 STEP 4 ─────── ─────── ─────── ─────── ☑ 남자복식 ☑ 오픈 (전연령) ● 전체 급수 ● 클럽 우선 ☑ 여자복식 ☑ 50대 (12명) ○ 개별 급수 ○ 급수 매칭 ☐ 혼합복식 ☑ 55대 (8명) ☐ S ☐ A ☐ B ... ○ 랜덤 ☑ 60대 (5명) ☐ 40대 (23명) 종목 유형 연령대 (인원수 표시) 급수 선택 팀편성 방식

사용 예시: 남복 + 여복 선택 → 50대 + 55대 선택 → 전체급수 → 클럽 우선
→ 자동으로 4개 종목 생성: 남자복식 50대 전체, 남자복식 55대 전체, 여자복식 50대 전체, 여자복식 55대 전체
→ 각 종목별 자동 팀편성까지 한번에 완료!

참가자 목록 연령대 표시

참가자 탭 상단에 연령대별 인원수 통계 배지가 표시되며, 참가자 목록 테이블에 연령대 컬럼이 추가되어 출생년도 기반으로 자동 계산된 연령대를 확인할 수 있습니다.

번호인쇄물내용사용 시점

③ 조별 대진표 라운드/코트/팀A vs 팀B, 승자 체크박스 (경기 미생성 시 빈 양식) 대진표 생성 후

사용법

인쇄 절차
  1. 홈 화면 → "인쇄 센터" 카드 클릭 (또는 대회 상세 → 상단 "인쇄" 버튼)
  2. 대회 선택 → 데이터 자동 로드 (6개 섹션 모두 표시)
  3. 상단 토글 버튼으로 필요한 섹션만 선택/해제 (클릭 시 해당 섹션으로 스크롤)
  4. "프린트" 버튼 클릭 → 브라우저 인쇄 대화상자 → "PDF로 저장" 또는 프린터 선택

팁: 인쇄 시 토글 버튼/컨트롤 패널은 자동으로 숨겨짐 (A4 용지만 출력)

⚠️ 주의사항
  • 대진표/점수기록지: 경기가 생성되지 않은 상태라면 빈 양식이 출력됩니다 (수기 기입용)
  • 결선 대진표: 조별 상위 2팀을 기입하면 브래킷이 완성됨 (크로스 시드: 1조 1위 vs 2조 2위)
  • 인쇄 타이밍: 대회 전날 대진표 생성 후 인쇄해두면 비상 시 즉시 사용 가능

잘못된 점수 입력 오입력 코트 점수판 Undo(되돌리기) 또는 본부에서 점수 수정
경기 순서 변경 필요 선수 부재 등 본부에서 해당 경기 → pending으로 되돌리고 다른 경기 시작
태블릿 배터리 방전 장시간 사용 보조배터리 연결 또는 예비 태블릿에 같은 URL 접속
인터넷 전체 끊김 공유기 장애 스마트폰 핫스팟으로 임시 전환 (모든 장비 재연결)
서명 화면 안 나옴 경기 미완료 상태 점수 목표점 도달 확인 → 경기 종료 흐름 재진행
대시보드 갱신 안 됨 브라우저 탭 비활성 탭 클릭하여 활성화 또는 새로고침
푸시 알림이 안 옴 브라우저 권한 미허용 브라우저 설정 → 사이트 알림 → "허용"으로 변경
iPhone은 "홈 화면에 추가" 후 재접속 필요
알림 구독 버튼 안 보임 이름 미검색 상태 /my 페이지에서 이름 입력 후 "조회" 먼저 클릭
iPhone에서 알림 불가 Safari 직접 접속 상태 Safari → 공유 → "홈 화면에 추가" → 홈 아이콘으로 재접속
(iOS 16.4 이상 필요)

9. 효율적 운영 팁

✅ 코트 자율 운영

locked=1&autonext=true 설정하면 코트 기록원만 있으면 본부 개입 없이 경기가 자동 순환됩니다.

✅ 경기 호출 타이밍

대시보드의 코트 현황에서 빈 코트가 보이면 즉시 다음 경기 팀을 방송 호출합니다.

✅ 결선 타이밍

모든 종목 예선 80%+ 완료 시점부터 완료된 종목부터 순차 결선 생성이 가능합니다.

✅ 관중 실시간 관람

코트 태블릿의 QR 버튼으로 관중이 스마트폰에서 해당 코트 실시간 점수를 볼 수 있습니다 (mode=view).

✅ 중간 점검

50% 진행률 시점에서 종목별 진행 상황을 확인하고 지연 종목을 우선 배정합니다.

✅ 분쟁 대비

모든 경기는 승/패팀 서명으로 확인됩니다. 중요 경기는 서명 화면 캡처를 권장합니다.

✅ 푸시 알림 적극 활용

참가자에게 /my 페이지에서 알림 구독을 안내하면 경기 호출 방송 횟수를 크게 줄일 수 있습니다. 경기 시작 시 해당 선수와 다음 대기 선수에게 자동으로 알림이 발송되므로, 운영 인력이 일일이 호출하지 않아도 됩니다.

  • 참가자 3~5명에게 알림 구독 테스트 (테스트 발송 버튼으로 확인)

🖨️ 인쇄 센터 (비상 대비)

  • 마이크/방송 장비 확인 (경기 호출용 — 알림 구독자 많으면 방송 최소화 가능)
